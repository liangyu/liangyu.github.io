2017.10.02 Liang Yu iOS  热度 ℃

iOS程序的完整启动过程

  1. 程序一开始会执行main函数：
    main(int argc, char* argv[]) { }

  2. 再执行UIApplicationMain 函数:
    UIApplicationMain(int argc, char argv[], NSString principalClassName, NSString* delegateClassName){ }

    • 2.1 根据参数principalClassName 创建 UIApplication对象
    • 2.2 根据参数delegateClassName 创建 UIApplication的delegate对象
  1. delegate对象开始处理监听事件
    • 3.1 没有storyboard
      • 3.1.1 程序启动完成后，会调用代理的 application: didFinishLaunchingWithOptions 方法
      • 3.1.2 在didFinishLaunchingWithOptions 中创建 UIWindow
      • 3.1.3 创建和设置UIWindow的rootViewController (根控制器)
      • 3.1.4 显示窗口
    • 3.2 根据Info.plist 获得最主要的storyboard的文件名，加载最主要的storyboard，然后步骤同无storyboard的情况。
      // 创建UIWindow
      // 创建和设置UIWindow的rootViewController
      // 显示窗口

UIApplication, AppDelegate, UIWindow, UIViewController 的关系

  • UIApplication对象内部有一个delegate属性，该属性就是AppDelegate文件
    • UIApplication对象，每一个应用都有，而且是单例
    • [UIApplication sharedApplication] 可以获取这个单例对象
    • UIApplication 能进行应用级别的管理，app启动后创建的第一个对象
  • AppDelegate对象内部有一个window属性，该属性就是创建出来的UIWindow
  • UIWindow对象内部有个rootViewController属性，该属性就是UIViewController
  • UIViewController内部又个view属性，该属性就是该控制器的UIView
