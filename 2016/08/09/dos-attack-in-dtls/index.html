What is the DTLS

DTLS is the abbreviation for Datagram Transport Level Security. DTLS provides a secure way to transport message in UDP method. It can prevent message from being eavesdropped(intercepted)，tampered orforgered. The DTLS is intended to provide the similar security guarantees as the TLS does.

If the four-way handshake is not implemented properly

Difference from TLS

Attachking demo

A DTLS session is usually established by using a four-way handshake

  1. Client sends the server a “Hello” packet with cookie length field set to 0.
  2. Server responds with a “Hello Verify” packet with a 16 bytes random cookie field.
  3. The client then resends the “Hello” packet incorporating the cookie value
  4. Server responds with a “Server Hello” message and completes the handshake.

Step #2 and #3 are there to prevent denial of service attacks against the platform. Because DTLS uses UDP, every ‘Hello’ message
can come from different source IP address which can be spoofed. This allows an attacker with a single computer to create thousands of
sessions on the server, each of which need to be tracked. This makes for an easy denial of service attack on the server which would be
extremely difficult to defend against. It is recommended that the DTLS server protocol is correctly implemented with the DOS protection detailsed in RFC 4347 Section 4.2.2

If the server does not send the “Hello Verify”, it might in the risk of DTLS server Dential of server attacks. The steps 2 and 3 are essential to prevent it from happening.

Reason:

  1. DTLS uses UDP, every Hello message can come from a different source IP address. And the address can be spoofed. If the bad guys create thousands of sessions to server from a single computer, the server is forced to track those tremendous numbers of sessions. Therefore, the server has a difficult to tell whether they are normal requests, or it is under attack.
